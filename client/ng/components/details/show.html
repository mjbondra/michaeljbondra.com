JavaScript

I am primarily a JavaScript developer. The hastily devised language is far from perfect — an obvious statement for those who have lived through the nightmares of loaded operators, unintuitive coercion, and 0.1 + 0.2 != 0.3. But a server-side runtime, evolving standards, and a myriad of excellent frameworks have given new life to the imperfect child of necessity.

The rise of Node.js has given JavaScript a reliable engine upon which complex applications can be built. The preeminent framework used for developing these applications is Express. It abstracts away the complexity of handling the request-response cycle with the powerful concept of sequentially executed functions known as middleware. The Koa framework will undoubtedly inherit the success of Express, and not just because they share an overlapping set of contributors. Koa extends the concept of middleware by layering functions known as generators, which can yield their execution to other generators, functions, and objects, before resuming their own.

On the front-end, JQuery has long been the standard by virtue of its simple, intuitive API for interacting with HTML documents. Utilities like Underscore and Lo-Dash have provided additional tools that can supplement any library or framework. But most exciting has been the advent of libraries and frameworks like AngularJS, Ember, Backbone.js, Flux, and React, with AngularJS and React having perhaps the greatest mindshare of this latest generation. AngularJS is complete framework that extends HTML as a templating language, with powerful two-way data-binding that synchronizes values between the document and the underlying JavaScript engine. React is an ultra-performative library that focuses on creating abstractions for interface functionality, and is often paired with another library or framework, namely Flux.
